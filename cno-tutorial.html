1. Introduction to Carbon Nano-Onions

Carbon nano-onions (CNOs) are spherical nanostructures composed of concentric fullerene shells. Think of them as Russian nesting dolls made of carbon atoms, where each layer is a fullerene cage nested inside another.

What are Fullerenes?

Fullerenes are molecules made entirely of carbon atoms arranged in a hollow sphere, ellipsoid, or tube. The most famous fullerene is C60 (buckminsterfullerene), which has 60 carbon atoms arranged in a soccer ball pattern with 12 pentagons and 20 hexagons.

Structure Formula

Number of atoms = 60 √ó n¬≤

Where n is the shell order (a positive integer):

Shell Order (n) Formula Atom Count Common Name
1 60 √ó 1¬≤ = 60 60 C60 (Buckminsterfullerene)
2 60 √ó 2¬≤ = 240 240 C240
3 60 √ó 3¬≤ = 540 540 C540
4 60 √ó 4¬≤ = 960 960 C960
5 60 √ó 5¬≤ = 1500 1500 C1500

Construction Methods

This tool uses two algorithms:

Goldberg-Coxeter (GC) Construction: A mathematical method for systematically creating icosahedral fullerenes. It places vertices on a sphere following specific symmetry rules.

Fibonacci-Guided Bonds: An enhancement that uses Fibonacci spiral distribution to guide bond formation, resulting in more uniform bond lengths and improved structural quality. This is the recommended method for research applications.

For most research purposes, use the "GC + Fibonacci Bonds" mode. It produces structures with more realistic geometries suitable for molecular dynamics simulations.

3. Parameter Reference

Shell Orders (n values)

Required: Yes
Format: Comma-separated positive integers
Example: 1,2,3 or 2,3,4

Defines which fullerene shells to include in your nano-onion. Each integer creates a shell with 60 √ó n¬≤ atoms.

Rules:

  • Must be positive integers (1, 2, 3, 4, ...)
  • No duplicates allowed
  • Order doesn't matter (the tool automatically sorts them)
  • Larger values = bigger shells with more atoms
Large structures (n > 10) can take several minutes to generate and may be slow to visualize. Start with smaller values (n ‚â§ 5) for testing.

Mode

Required: Yes
Options: "Goldberg-Coxeter (n,0)" or "GC + Fibonacci Bonds"
Recommended: GC + Fibonacci Bonds

Goldberg-Coxeter (n,0): Standard icosahedral construction. Fast but produces less uniform bond lengths.

GC + Fibonacci Bonds: Enhanced algorithm that uses Fibonacci sphere distribution to guide bond formation. Results in:

  • More uniform bond length distribution
  • Better suited for molecular dynamics
  • More realistic fullerene geometries
  • Slightly slower generation time

Random Seed

Required: No (optional)
Format: Integer
Example: 42, 12345, 987654

Seeds the random number generator for reproducible results. Using the same seed with identical parameters guarantees the exact same structure.

When to use:

  • Publishing research (include seed in methods for reproducibility)
  • Comparing different parameter sets
  • Debugging or testing

When to leave blank:

  • Exploring different structural variations
  • Initial testing and prototyping

Vacuum Padding (√Ö)

Required: No (optional, default: 0)
Format: Number ‚â• 0
Typical Range: 10-20 √Ö
Example: 10.0

Adds empty space around the structure in exported coordinate files. This is crucial for periodic boundary condition simulations.

Why it matters:

  • Prevents artificial interactions between periodic images
  • Allows proper solvation shell formation in explicit solvent simulations
  • Reduces edge effects in finite-temperature simulations

Recommended values:

  • 10-15 √Ö: Gas phase or implicit solvent MD
  • 15-20 √Ö: Explicit solvent simulations
  • 0 √Ö: Visualization only, no simulations planned

Bond Scale

Required: No (default: 1.00)
Format: Number > 0
Typical Range: 0.8-1.2
Example: 1.00

Controls the strength of bond regularization during structure optimization. Higher values push bonds closer to the ideal sp¬≤ carbon-carbon distance (~1.42 √Ö).

Effect on structure:

  • 0.8-0.95: Softer constraints, more geometric flexibility, slightly irregular bonds
  • 1.00: Balanced (recommended default)
  • 1.05-1.2: Stricter regularization, more uniform bonds, less geometric freedom
Start with 1.00. Only adjust if bond-length histogram shows poor distribution. Most users never need to change this parameter.

Reference Shell File

Required: No (optional)
Format: Plain text file with "C x y z" lines
Example: optimized_c240.xyz

Upload a previously optimized fullerene structure to automatically calibrate the Œ± (alpha) parameter. The tool measures the radius of your reference and scales all shells proportionally.

Use cases:

  • You have DFT-optimized fullerene structures
  • Matching experimental or literature structures
  • Ensuring consistency across multiple CNO variants

File format:

C 0.000000 0.000000 1.420000
C 1.229321 0.000000 0.710000
C 0.614661 1.064638 0.710000
C -0.614661 1.064638 0.710000
...

Each line must start with "C" followed by x, y, z coordinates in √Öngstr√∂ms.

When using a reference file, you MUST also specify "Reference Shell n_ref" to indicate which shell your reference corresponds to.

Reference Shell n_ref

Required: Only if using Reference Shell File
Format: Integer matching one of your shell orders
Example: 2 (if reference is C240)

Tells the tool which shell your reference file represents. For example:

  • If your reference has 60 atoms (C60), set n_ref = 1
  • If your reference has 240 atoms (C240), set n_ref = 2
  • If your reference has 540 atoms (C540), set n_ref = 3

The tool will:

  1. Measure the mean radius of your reference structure
  2. Calculate Œ± = measured_radius / n_ref
  3. Use this Œ± to scale all other shells proportionally

Alpha (√Ö)

Required: Yes (unless using Reference File)
Format: Number > 0
Typical Range: 3.4-3.7 √Ö
Recommended: 3.54 √Ö

The scaling factor that determines shell radii. Each shell's radius is approximately Œ± √ó n.

Physical meaning: Œ± roughly corresponds to the radial distance per shell order, related to the inter-shell spacing and carbon-carbon bond length.

Common values:

  • 3.45 √Ö: Tighter packing, shells closer together
  • 3.54 √Ö: Standard value, well-balanced (recommended)
  • 3.65 √Ö: Looser packing, more inter-shell space
If you upload a reference file and leave Œ± blank, it will be automatically calculated. This is the most accurate way to set Œ± for structures matching literature or experimental data.

Output Prefix

Required: No (auto-generated if blank)
Format: Text string
Example: my_cno, onion_123, test_structure

Sets the base filename for all exported files. If you enter "my_cno", you'll get:

  • my_cno.xyz - XYZ coordinate file
  • my_cno.cif - CIF crystallographic file
  • my_cno.data - LAMMPS data file
  • in.my_cno - LAMMPS input script

If left blank, the tool auto-generates a descriptive name like onion_1_2_3 based on your shell orders.

4. Visualization & Analysis

3D Viewers

The tool provides two complementary 3D visualizations:

WebGL Viewer (Top): Custom-built renderer showing atoms as colored balls and bonds as lines. Fast and responsive for real-time interaction.

Crystal Toolkit Viewer (Bottom): Professional-grade molecular visualization using Materials Project's library. Provides advanced rendering with lighting and materials.

Mouse Controls

  • Left-click + drag: Rotate structure in 3D
  • Scroll wheel: Zoom in/out
  • Double-click (Crystal Toolkit): Reset view to default

Display Options

Show bonds: Toggle chemical bond lines between carbon atoms. Useful for analyzing connectivity and identifying defects.

Show axes: Display X/Y/Z coordinate axes for orientation. Red = X, Green = Y, Blue = Z.

Shell view dropdown: Filter visualization to show:

  • All shells: Complete nano-onion structure
  • Individual shells: Isolate specific shells (e.g., only C60, only C240)

Color Legend

Each shell is assigned a unique color (blue ‚Üí purple gradient). Click legend items to quickly filter to that shell.

Bond-Length Histogram

Displays the distribution of distances to nearest neighbors. This is crucial for quality assessment:

1st Nearest Neighbors (1st NN):

  • Direct chemical bonds
  • Should peak around 1.42 √Ö for sp¬≤ carbon
  • Narrow distribution indicates uniform bond lengths

2nd Nearest Neighbors (2nd NN):

  • Atoms across hexagon edges
  • Typically ~2.4-2.5 √Ö
  • Helps identify hexagon/pentagon regularity

3rd Nearest Neighbors (3rd NN):

  • Further structural correlations
  • Useful for analyzing long-range order

NN Inspector Metrics

Displays min/mean/max values for the selected neighbor order:

Good quality indicators:

  • 1st NN mean close to 1.42 √Ö (¬±0.05 √Ö)
  • Narrow distribution (max - min < 0.2 √Ö for 1st NN)
  • Symmetric histogram shape
If your 1st NN histogram shows multiple peaks or very wide distribution, try adjusting Bond Scale or using Fibonacci mode for better regularization.

5. Export Formats

XYZ Format

Use for: General molecular visualization, quick analysis

Compatible with: VMD, Avogadro, PyMOL, Jmol, most molecular viewers

File structure:

[number of atoms]
[comment line with metadata]
C  x1  y1  z1
C  x2  y2  z2
...

Advantages:

  • Simple, human-readable format
  • Universal compatibility
  • Small file size

CIF Format (Crystallographic Information File)

Use for: Crystallographic databases, materials science applications

Compatible with: Materials Project, COD, VESTA, Mercury, CrystalMaker

Features:

  • Includes lattice parameters
  • Fractional coordinates
  • Symmetry information
  • Database-ready format

Advantages:

  • Standard in materials science
  • Preserves unit cell information
  • Suitable for publication and archiving

LAMMPS Format

Use for: Molecular dynamics simulations

Compatible with: LAMMPS molecular dynamics package

Files generated:

  1. prefix.data - LAMMPS data file with atomic coordinates and topology
  2. in.prefix - Ready-to-run LAMMPS input script

Included in input script:

  • AIREBO potential for carbon (accurate for sp¬≤ systems)
  • Initial thermalization at 5 K
  • 5000-step MD equilibration
  • Conjugate gradient energy minimization
  • Output of relaxed structure

To run simulation:

lmp -in in.my_cno
The provided LAMMPS script is a starting template. Adjust temperature, timestep, and run length based on your specific research needs. Always test with short runs first.

Which Format Should I Use?

Purpose Recommended Format
Quick visualization XYZ
Molecular dynamics LAMMPS
Database submission CIF
Materials science analysis CIF
Quantum chemistry (Gaussian, ORCA) XYZ (then convert)
Publication supplement XYZ + CIF

6. Example Workflows

Example 1: Basic 3-Shell Nano-Onion

Goal: Create a standard C60@C240@C540 nano-onion for visualization

Parameters:

  • Shell Orders: 1,2,3
  • Mode: GC + Fibonacci Bonds
  • Alpha: 3.54
  • Bond Scale: 1.00
  • Vacuum Padding: 0 (visualization only)

Result: 840 atoms (60 + 240 + 540) in ~10 seconds

This is the recommended starting configuration for first-time users. It generates quickly and produces a clean, easy-to-visualize structure.

Example 2: MD Simulation Preparation

Goal: Prepare a CNO for LAMMPS molecular dynamics at finite temperature

Parameters:

  • Shell Orders: 2,3,4
  • Mode: GC + Fibonacci Bonds
  • Alpha: 3.54
  • Bond Scale: 1.00
  • Vacuum Padding: 15.0
  • Random Seed: 42 (for reproducibility)
  • Output Prefix: cno_234_md

Workflow:

  1. Generate structure with above parameters
  2. Download LAMMPS format
  3. Modify in.cno_234_md to set desired temperature and run time
  4. Run: lmp -in in.cno_234_md
  5. Analyze trajectory with OVITO or VMD

Example 3: Reference-Based Generation

Goal: Create CNO matching a DFT-optimized C240 reference structure

Steps:

  1. Optimize C240 with DFT (Gaussian, ORCA, VASP, etc.)
  2. Export optimized geometry as XYZ format
  3. In CNO Generator:
    • Shell Orders: 1,2,3,4
    • Mode: GC + Fibonacci Bonds
    • Reference File: Upload your optimized C240 XYZ
    • Reference n_ref: 2
    • Alpha: leave blank (auto-calculated)
    • Vacuum Padding: 10.0
  4. Generate structure

Result: All shells scaled to match your reference structure's dimensions

This approach ensures consistency when comparing computational results across different CNO sizes. The reference-based Œ± ensures all shells use the same radial scaling.

Example 4: Large Structure for Visualization

Goal: Create an impressive 5-shell nano-onion for presentations

Parameters:

  • Shell Orders: 1,2,3,4,5
  • Mode: GC + Fibonacci Bonds
  • Alpha: 3.54
  • Bond Scale: 1.05 (slightly tighter bonds)
  • Vacuum Padding: 0

Result: 3,300 atoms total (60 + 240 + 540 + 960 + 1500)

Generation time: 1-3 minutes. Visualization may be slow on older hardware. Use shell filtering to view individual layers for better performance.

Example 5: Bond Quality Optimization

Goal: Fine-tune bond uniformity for critical applications

Iterative approach:

  1. Generate with default Bond Scale (1.00)
  2. Check 1st NN histogram - note distribution width
  3. If bonds too irregular (range > 0.15 √Ö):
    • Increase Bond Scale to 1.10
    • Regenerate and compare histogram
  4. If bonds too constrained (mean drifts from 1.42 √Ö):
    • Decrease Bond Scale to 0.95
    • Regenerate and compare
  5. Iterate until satisfied with bond distribution

Target metrics:

  • 1st NN mean: 1.40-1.44 √Ö
  • 1st NN range (max - min): < 0.12 √Ö
  • Histogram shape: Single narrow peak

7. Troubleshooting

Problem: "Shell list looks good" but Generate button doesn't work

Solution:

  • Check that all parameters are valid (look for red error messages)
  • Ensure Alpha is set if not using a reference file
  • Verify Vacuum Padding and Bond Scale are positive numbers

Problem: Generation takes too long / browser freezes

Causes:

  • Too many shells or very large n values (n > 10)
  • Older/slower hardware

Solutions:

  • Start with fewer, smaller shells (e.g., 1,2,3)
  • Use standard GC mode instead of Fibonacci (faster but lower quality)
  • Close other browser tabs to free memory
  • Be patient - structures with 5000+ atoms can take 2-5 minutes

Problem: Bond histogram shows multiple peaks

Causes:

  • Bond Scale too low, allowing irregular geometry
  • Very large shells with difficult topology

Solutions:

  • Increase Bond Scale to 1.10-1.15
  • Switch to Fibonacci mode if using standard GC
  • Try different Random Seed values

Problem: Crystal Toolkit viewer not showing structure

Solutions:

  • Refresh the page and regenerate
  • Check browser console for errors (F12)
  • The WebGL viewer should still work - use that instead
  • Try a different browser (Chrome/Firefox recommended)

Problem: Reference file upload fails

Common issues:

  • "No C x y z lines found" - File format incorrect. Ensure each line starts with "C" followed by three numbers
  • n_ref doesn't match shells - Your reference must correspond to one of your shell orders
  • File too large - References should be single shells, not complete CNOs

Valid reference line examples:

C 0.000 0.000 1.420
C  1.229  0.000  0.710
C    0.615    1.065    0.710

Problem: LAMMPS simulation fails immediately

Solutions:

  • Ensure AIREBO potential file (CH.airebo-m) is in same directory
  • Check that LAMMPS is compiled with MANYBODY and MOLECULE packages
  • Verify file paths in input script are correct
  • Try running with fewer timesteps first (change "run 5000" to "run 100")

Problem: Visualization is laggy/slow

Solutions:

  • Uncheck "Show bonds" to reduce rendering load
  • Use shell filtering to view one shell at a time
  • Generate smaller structures for testing
  • Close other applications to free GPU/memory

Problem: Downloaded file won't open in my software

Format-specific solutions:

XYZ files:

  • Try opening in text editor first to verify format
  • Some software requires .xyz extension - check filename
  • VMD: Use "New Molecule" ‚Üí "Browse" ‚Üí select file
  • Avogadro: File ‚Üí Open ‚Üí select XYZ from file type dropdown

CIF files:

  • VESTA: File ‚Üí Open ‚Üí select .cif file
  • Mercury: File ‚Üí Open Structure ‚Üí browse to .cif
  • Verify file isn't corrupted by opening in text editor

LAMMPS files:

  • Both .data and in.* files must be in same directory
  • Check that paths in input script match your filenames
  • Ensure potential file is available

Still Having Issues?

If problems persist:

  1. Try the "Reset" button and start fresh
  2. Test with the Example 1 workflow (known working configuration)
  3. Clear browser cache and reload page
  4. Try a different browser
  5. Check browser console (F12) for error messages
When reporting issues, include: browser version, shell orders used, all parameter values, and any error messages from console.
